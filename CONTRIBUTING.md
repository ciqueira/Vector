# Contributing to Vector

Vector is a public source-available project with compiled commercial
distribution. It is not an open-source project.

Bug reports, test results, documentation corrections, attribution evidence,
and focused code contributions are welcome. Please open an issue before
submitting a substantial implementation.

## Contributor Rights

Only submit material that you authored or that you have the legal right to
submit under these terms. Do not submit code copied from tutorials,
repositories, papers, DCTLs, plugins, SDK examples, or AI-generated output
unless its provenance and license have been reviewed and documented.

By intentionally submitting a pull request or other contribution for inclusion
in Vector, you represent that you have the authority to do so and grant Magno
Ciqueira a perpetual, worldwide, non-exclusive, royalty-free, irrevocable
license to use, reproduce, modify, prepare derivative works from, publicly
display, compile, distribute, sublicense, and commercialize the contribution in
source and binary form.

You retain ownership of your contribution. This grant allows the project to
include it in public source-available code and commercial OFX releases without
requiring the entire project to adopt an open-source license.

If you cannot grant these rights, do not submit the contribution. A separate
written agreement may be arranged before submission.

## Required Pull-Request Information

Every pull request should state:

- what was changed and why;
- whether the contribution is entirely original;
- all external references, specifications, papers, repositories, or code used;
- the license of every external source;
- how the change was tested.

Contributions affecting color processing should include reference values or
visual comparisons where practical.

## Licensing and Provenance Reports

If you are an author or rights holder for material identified in
`THIRD_PARTY_NOTICES.md`, use the licensing/provenance issue form. Permission
must be explicit about public source display, modification, compilation,
binary distribution, and commercial use.

Maintainers may preserve authorization records privately when they contain
personal contact information, while recording the scope and date of the grant
in `THIRD_PARTY_NOTICES.md`.
